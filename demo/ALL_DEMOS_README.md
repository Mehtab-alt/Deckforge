# DeckForge Complete Interactive Platform Demos

This folder contains multiple interactive demonstrations of the complete DeckForge platform evolution through all 10 phases.

## Available Demos

### 1. Comprehensive Platform Demo
- **File**: `comprehensive_demo.py`
- **Description**: Complete walkthrough of all 10 phases in sequence
- **Features**: Shows the entire evolution story from static generator to autonomous SRE system
- **Run**: `python demo/comprehensive_demo.py`

### 2. Interactive Phase Explorer
- **File**: `interactive_comprehensive_demo.py`
- **Description**: Interactive menu-driven exploration of each phase
- **Features**: 
  - Choose specific phases to explore
  - Configure parameters for each phase
  - Complete platform journey simulation
  - Architecture overview and business impact
  - Setup and configuration guide
- **Run**: `python demo/interactive_comprehensive_demo.py`

### 3. Visual SRE Demo
- **File**: `visual_demo.py`
- **Description**: Visual simulation of Phases 9-10 (Autonomous SRE & Multi-Cloud Federation)
- **Features**: Animated visualization of incident detection, investigation, and remediation
- **Run**: `python demo/visual_demo.py`

### 4. Interactive SRE Demo
- **File**: `interactive_demo.py`
- **Description**: Interactive version allowing custom scenarios for SRE features
- **Features**: 
  - Input your own infrastructure scenarios
  - Learn how DeckForge works internally
  - View configuration options
  - Understand user workflows
- **Run**: `python demo/interactive_demo.py`

## Platform Evolution Overview

### Phases 1-4: Static Infrastructure Generation
- AI-powered blueprint creation from user ideas
- Multi-modal vision processing for diagrams
- Knowledge base integration for policy compliance
- Template rendering for Terraform/Ansible

### Phase 5: Infrastructure State Tracking
- Real-time monitoring of deployed resources
- Live state synchronization
- Resource health tracking

### Phase 6: Role-Based Access Control
- User authentication and authorization
- Permission management for different roles
- Security enforcement

### Phase 7: Asynchronous Workers
- Celery-based task processing
- Long-running infrastructure generation
- Distributed workload handling

### Phase 8: Continuous Learning
- AI feedback loop improvement
- Pattern recognition in infrastructure requests
- Model training from user corrections

### Phase 9: Autonomous SRE
- Incident detection and ingestion
- Automated investigation via SSH diagnostics
- AI decision engine for remediation
- Slack integration for human approval
- Ansible runbook execution

### Phase 10: Multi-Cloud Federation
- Cross-cloud resource abstraction
- Provider-agnostic infrastructure intents
- Cloud-specific SKU mapping
